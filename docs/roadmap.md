# Ideas Studio — Roadmap future

## Objectif

Cette roadmap garde les idées produit importantes pour l'après-V1. Elle ne décrit pas des fonctionnalités déjà disponibles et ne doit pas déclencher de développement immédiat.

La V1 reste centrée sur le workflow éditorial : projets, idées, articles, analyse, publication, performance et optimisations.

## Non inclus dans la V1 actuelle

Ces sujets sont à prévoir plus tard. Ils demandent une décision produit, une conception backend, une intégration externe ou une vraie phase design avant développement.

| Sujet | Objectif | Notes |
|---|---|---|
| Site marketing / landing page | Présenter Ideas Studio avant inscription | Accueil, fonctionnalités, tarifs, FAQ, témoignages, CTA créer un compte / se connecter / voir démo |
| Pricing / paiement / abonnements | Monétiser le SaaS proprement | Plans, paiement, abonnement actif/expiré, relance paiement échoué, limites par plan |
| Dashboard admin interne | Administrer Ideas Studio côté équipe interne | Utilisateurs, projets, paiements, erreurs, connexions sites, usage IA, support, logs |
| Support / Help | Aider les utilisateurs sans bloquer le produit | Centre d'aide, FAQ, contact support, chat support plus tard |
| Inbox / formulaires de contact | Recevoir les messages des blogs connectés dans Ideas Studio | Messages, réponses, lu/non lu, filtres projet/site, notifications |
| Newsletter | Gérer les abonnés et campagnes depuis Ideas Studio | Abonnés, éditeur, personnalisation, stats ouverture/clics, désabonnement, segmentation |
| Chat équipe | Centraliser les échanges projet | Discussion interne, historique, notifications |
| Pages légales | Préparer la mise en production publique | Confidentialité, CGU, mentions légales, cookies |
| Couleur de marque | Définir une identité propre Ideas Studio | Remplacer le bleu générique plus tard, garder une base blanc/noir/neutres |

## Rappels produit

- Ne pas afficher ces fonctionnalités comme si elles étaient déjà disponibles.
- Utiliser "Bientôt disponible" ou "Dépend du backend" quand l'action n'est pas branchée.
- Ne pas simuler de réussite si rien n'est réellement persisté.
- Garder Help visible dans le menu quand la route existe.
- Garder Déconnexion facile à trouver dans la sidebar ou le profil.
- Garder Générer dans la section Intelligence.
